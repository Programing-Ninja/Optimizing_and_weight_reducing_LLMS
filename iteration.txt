--- EXPERIMENT FILE ---

File: /scratch/DA24B039/sct_tq/experiments/sct_tq_pareto.py

What it does:
  Sweeps SCT energy {None=dense, 0.99, 0.95, 0.90, 0.80} x TurboQuant KV configs
  {fp16-full, (k=4,v=4), (k=3,v=4), (k=3,v=2), (k=2,v=2)} on SmolLM2-135M.

  SCT: replaces MLP nn.Linear layers (gate_proj/up_proj/down_proj) with
  SpectralLinear.from_linear() at the given energy threshold. This does a
  truncated SVD and stores W = U diag(s) V^T. At 135M scale this rarely
  compresses — see COUNTERINTUITIVE section below.

  KV compression: uses a DynamicLayer subclass (TurboQuantLayer) that intercepts
  the transformers 5.x cache layer update() call, compresses keys via
  TurboQuantProd and values via quantize_values (group quantization), stores
  compressed tensors, and returns dequantized full-history K and V for attention.
  The Cache subclass approach WORKED — no fallback needed.

  Per config: measures perplexity, weight bytes, compressed KV bytes, peak RSS,
  and tokens/sec. Computes Pareto frontier (non-dominated on all 4 axes).
  Saves results to experiments/sct_tq_pareto_results.json.
  Saves scatter plot to experiments/sct_tq_pareto.png.

--- WHAT DIDN'T WORK / COUNTERINTUITIVE FINDINGS ---

1. SCT gives NEGATIVE weight compression on SmolLM2-135M at E=0.95:
   MLP params 79.6M -> 84.3M (0.94x, i.e. 6% LARGER). This confirms the
   CLAUDE.md warning: hidden_dim=576 << 2048; MLP layers are too small for
   SVD factors to compress. The SVD adds U, s, V overhead that exceeds savings.

2. SCT severely degrades perplexity post-hoc (without finetuning):
   22.76 -> 16945 at E=0.95 (without finetune). This is expected — applying
   SVD truncation without finetune to a converged model destroys quality.
   The --finetune-steps flag would recover quality but was not tested.

3. TurboQuant perplexity impact is larger than expected at 128 tokens:
   This is because for short sequences, every key gets compressed (no buffer).
   At longer sequences (768+ tokens) the PPL impact should be smaller in relative
   terms since early tokens matter less for the later predictions.

4. Wikitext-2 dataset loading fails (HF URI format changed in datasets 5.x):
   "Invalid HF URI 'hf://datasets/wikitext@b08601e04326c79dfdd32d625aee71d232d685c3/...'"
   Fallback to hardcoded English text works; results are internally consistent.

5. TurboQuant latency INCREASES vs dense baseline (1400 tok/s -> 702 tok/s):
   As documented in CLAUDE.md, dequantizing history to float32 on every decode
   step adds overhead. CPU TurboQuant decode is slower than plain fp16 matmul.

=== [2026-06-15] TASK: Per-component LR finetune + HP-tuning harness ===

What was implemented:
  experiments/sct_finetune.py:
    - finetune_sct() function: two AdamW param groups (SCT factors at sct_lr,
      dense group at dense_lr = sct_lr / lr_ratio), cosine schedule w/ short
      warmup (min(20, steps//5)), grad clip 1.0, retract_all after every step.
    - Param groups: walk model.modules() for SpectralLinear -> collect m.U, m.s,
      m.V (group B, high lr). For group A: norm layers (all unfreeze modes) +
      q/k/v/o_proj linear layers (unfreeze=="norms+attn" only).
    - Data: tatsu-lab/alpaca, 500 samples, seq 128, formatted as alpaca template.
    - Returns {final_loss, final_ppl, steps, time_sec, sct_params, dense_params,
      loss_curve}.

  experiments/sct_hp_tune.py:
    - CLI: --energies, --sct-lrs, --lr-ratios, --unfreeze, --steps,
      --max-eval-tokens, --probe.
    - Grid sweep: for each config: load fresh model, apply SCT (adaptive rank via
      SVD), measure pre-SCT ppl, measure post-SCT ppl, call finetune_sct(),
      measure post-finetune ppl on wikitext-2 eval text. try/except per config.
    - Results table sorted by ppl; saves JSON to sct_hp_tune_{model_tag}.json.
    - --probe mode: load model, apply SCT, run 3 steps, report RAM delta and
      per-step timing with projections for 200/300/500 steps.

  experiments/sct_tq_pareto.py (Task C wire-in):
    - Added --sct-lr (default 5e-4), --lr-ratio (default 25), --unfreeze (default
      norms+attn) argparse args.
    - Replaced stub "finetune skipped" comment with real finetune_sct() call.
    - Lazy import via _get_finetune_fn() so no breakage when --finetune-steps==0.

Bug fixed during implementation:
  - Installed spectral_compact_training.SpectralLinear.from_linear() takes `rank`
    only (no energy_threshold). Implemented _adaptive_rank() helper (SVD-based)
    in sct_hp_tune.py and sct_finetune.py to compute k from energy threshold,
    then pass explicit rank to from_linear().

What didn't work / caveats:
